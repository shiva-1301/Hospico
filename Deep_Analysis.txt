# HospiCo — Complete Project Deep Analysis

A comprehensive analysis of the **HospiCo** healthcare platform — a full-stack hospital finder and booking application.

| Layer | Technology | Details |
|:------|:-----------|:--------|
| **Frontend** | React 19 + TypeScript + Vite | SPA with MUI, Tailwind, Framer Motion |

| **Database** | MongoDB (primary) | Collection-based, via MongoTemplate |
| **AI Chatbot** | Groq LLM API | Symptom analysis, hospital recommendations |
| **Mobile** | Capacitor 7.x | Hybrid mobile app wrapping the React SPA |
| **Deployment** | Render / Zoho Catalyst | Docker, Nginx, AppSail |

### 3.2 Service Layer — Strategy Pattern

The backend uses a **provider-switchable** data layer:

```mermaid
graph LR
  subgraph Interfaces
    DS["DataStoreService"]
    US["UserStoreService"]
  end
  subgraph "Mongo Provider"
    MDS["MongoDataStoreService"]
    MUS["MongoUserService"]
  end
  subgraph "Zoho Provider"
    ZDS["ZohoDataStoreService"]
    ZUS["ZohoUserService"]
  end
  DS --> MDS
  DS --> ZDS
  US --> MUS
```

### 3.4 Security & Auth

- [SecurityConfig](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/java/com/hospitalfinder/backend/config/SecurityConfig.java) — Stateless JWT sessions, CORS configurable via `ENABLE_SPRING_CORS` env var
- [JwtAuthenticationFilter](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/java/com/hospitalfinder/backend/filter/JwtAuthenticationFilter.java) — Extracts `Bearer` tokens, validates via [JwtService](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/java/com/hospitalfinder/backend/service/JwtService.java#16-74)
- **Public endpoints**: auth, clinics, appointments, chat, health, swagger
- **Protected endpoints**: `/api/users/**` (authenticated only)

### 3.5 AI Chatbot (ChatController — 717 lines)

The largest controller implements an **AI-powered medical assistant**:

- **LLM Integration**: Groq API (`groq.api.key` env var)
- **Symptom Analysis**: Detects symptom keywords → sends to LLM for specialization mapping
- **Hospital Recommendations**: Matches specializations to clinic database, geo-sorted
- **Multi-step Booking Flow**: Via [ChatSession](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/java/com/hospitalfinder/backend/entity/ChatSession.java#13-50) state machine (symptom → hospital → doctor → date → time → confirm)
- **Multi-language Support**: Language code detection + translation
- **Fuzzy Matching**: Levenshtein distance for specialization normalization

### 3.6 Configuration

[application.yml](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/resources/application.yml):
- **MongoDB**: `MONGODB_URI` (default: `mongodb://localhost:27017/HospiiCo`)
- **Server Port**: `8080` (configurable via `PORT` or `X_ZOHO_CATALYST_LISTEN_PORT`)
- **Data Provider**: `DATA_STORE_PROVIDER` (default: `mongo`, option: `zoho`)
- **Groq API**: `GROQ_API_KEY` for AI chatbot
- **Zoho Catalyst**: Project ID, Client ID/Secret, Refresh Token

## 6. Key Observations & Notes

> [!IMPORTANT]
> The project has **migrated from PostgreSQL to MongoDB** but the [docker-compose.yml](file:///c:/Users/chunk/Desktop/Qod/Hospico/docker-compose.yml) and [tech_info](file:///c:/Users/chunk/Desktop/Qod/Hospico/tech_info) documentation still reference PostgreSQL. The actual active database is MongoDB as confirmed by [application.yml](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/resources/application.yml) and the `spring-boot-starter-data-mongodb` dependency.

> [!WARNING]
> The JWT secret key is **hardcoded** in [JwtService.java](file:///c:/Users/chunk/Desktop/Qod/Hospico/backend/src/main/java/com/hospitalfinder/backend/service/JwtService.java). This should be moved to an environment variable for production security.

> [!NOTE]
> The `chabtot_Relevent/` directory contains reference/backup copies of chatbot-related files used during development. These are not part of the active codebase.

### Architecture Highlights
- **Strategy Pattern** for data storage — clean swap between MongoDB and Zoho Catalyst
- **AI-powered chatbot** with multi-step conversational booking
- **Capacitor** integration enables wrapping the React SPA as a native Android app
- **Offline support** with [ConnectivityHandler](file:///c:/Users/chunk/Desktop/Qod/Hospico/hospico-frontend-main/src/App.tsx#130-153) component
- **Dark mode** via `ThemeContext`
- **Keep-alive hook** (`useKeepAlive`) to prevent Render free-tier cold starts
